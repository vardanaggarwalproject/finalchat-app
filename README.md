## Overview

In traditional applications, communication happens using **HTTP-based web transporters**, which are not ideal for real-time interactions.

Today, technologies like **Socket.IO** upgrade this experience by enabling:

- Full-duplex communication  
- Bi-directional data transfer  
- Real-time message delivery  
- Minimal latency for instant updates  

This project demonstrates these concepts through a fully functional real-time chat system.

---

##  Features

### Real-Time One-to-One Messaging
Instant delivery of messages between two users using Socket.IO.

###  Group Chat via Rooms
- Each group chat is implemented using **Socket.IO rooms**.  
- Messages are broadcast only to members of that room.  
- All room participants immediately receive new messages.

###  Real-Time User Profile Updates
Users can update their profile, and changes automatically reflect across connected clients.

###  Live Active / Inactive User Status
The system shows the user's real-time presence:
- Online  
- Offline
- Active  
- Inactive
